# Contributing

Vibe Launch Review should remain a narrow pre-launch safety gate for vibe-coded products.

## Scope Rules

Accept contributions that improve:

- The five review gates.
- Evidence quality.
- Report clarity.
- Language matching.
- Boundary enforcement that prevents code fixes or broad pentest drift.

Do not expand this skill into:

- A general application security scanner.
- A penetration-testing playbook.
- An implementation or patch-generation agent.
- A service-integration guide.

## Adding Evals

New evals should include:

- A realistic user prompt.
- Expected output behavior.
- Assertions for verdict, scope, evidence handling, and no-fix behavior.

Use mock products and mock data only.
